# Milestone v2.0: Modular CPU-Only Platform

**Status:** ✅ SHIPPED 2026-04-15
**Phases:** 8-13
**Total Plans:** 16

## Overview

Make the platform modular — transcription works standalone on 16GB CPU, with diarization and LLM summaries as optional add-ons. Research and integrate better/lighter models for CPU-only use. Replace all infrastructure services with embedded alternatives. Package as a one-click Electron desktop app for non-technical users on air-gapped Windows machines.

## Milestone Summary

**Key Decisions:**
- large-v3-turbo replaces distil-large-v3.5 — all distil-whisper models are English-only
- Direct Java if/else replaces Conductor SWITCH — no workflow engine library needed
- HTTP retry (3 retries, 2s backoff) replaces Conductor's transparent retry
- JSON CLOB columns via @Convert for nested DOs — simpler than @ElementCollection
- File-based H2 (jdbc:h2:file:./data/blackbox-db) for persistent data across restarts
- docker-compose.yml deleted — all 7 infrastructure services eliminated
- Dynamic import for pdfmake — PDF chunks only loaded on export
- Custom app:// protocol for serving SPA dist/ in Electron
- resolveRuntimePaths() fallback — bundled paths preferred, system PATH fallback for dev
- CPU-only PyTorch in conda-pack (~2-3GB vs ~5GB with CUDA)
- @Lazy on AnalysisService to break circular bean dependency

**Issues Resolved:**
- Conductor + 6 infrastructure services fully eliminated
- WhisperX replaced with faster-whisper for CPU optimization
- Pipeline mode detection and graceful degradation implemented
- 15+ runtime issues fixed during integration testing
- Circular dependency between AnalysisServiceImpl and PipelineOrchestrationService

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- Placeholder icons for Electron app (need real branding)
- Windows NSIS installer not yet tested on target machine
- Full pipeline LLM validation deferred (Ollama unavailable during testing)
