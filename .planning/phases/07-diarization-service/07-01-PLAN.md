**Data flow understanding:**
1. Diarization outputs fragments with speaker="Speaker 00/01/02"
2. Workflow task `determine_speakers` (LLM) extracts real names → saved to `determinedSpeakers`
3. Workflow task `determine_rounds` (LLM) identifies game rounds → saved to `determinedRounds`
4. Workflow task `execute_prompts` generates insights per round → saved to `promptResponses`
5. Frontend translates speaker names via `determinedSpeakers` mapping
6. Frontend displays prompts via `determinedRounds[].promptResponses`

**Issue from CONTEXT.md:**
- Transcription works but speakers aren't named
- Summarization doesn't appear
- Need to trace where data stops flowing
