<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Configuration audit complete. Now validating transcription-only mode end-to-end — the simplest pipeline mode with fewest dependencies.</what-built>
  <how-to-verify>
**Start infrastructure and backend (do NOT start diarization or chat services):**

1. Start Docker infrastructure:
   ```bash
   cd blackbox && docker compose up -d
   ```
   Wait ~30s for all 7 services (MongoDB, Conductor, MinIO, etc.) to be healthy.

2. Start the Spring Boot backend:
   ```bash
   cd blackbox/backend && JAVA_HOME=/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home mvn spring-boot:run -pl blackbox_application -Dspring-boot.run.profiles=local
   ```
   Wait for "Started Application" log line.

3. Start the frontend:
   ```bash
   cd blackbox/frontend && npx quasar dev
   ```
   Wait for dev server to start on port 9003.

**Validate pipeline status (transcription-only auto-detection):**

4. Open http://localhost:9003 in browser
5. Check AppBar service status chips:
   - Diarization chip should be **grey** with hollow circle icon
   - Chat chip should be **grey** with hollow circle icon
   - Mode chip should show **"Mode: Transcription Only"**
   - Hovering grey chips should show **"Service offline"** tooltip

6. Verify the API directly:
   ```bash
   curl -s http://localhost:8081/pipeline/status | python3 -m json.tool
   ```
   Expected: `resolvedMode: "transcription_only"`, `configuredMode: "auto"`, both services false

**Run transcription-only workflow:**

7. Create or select a workspace in the UI
8. Upload a test audio file (use an existing file from v1.0 testing)
9. Click "Run" / start analysis
10. Verify:
    - Processing starts without errors
    - Conductor workflow executes (check http://localhost:5001 for workflow status)
    - Transcription completes with text output
    - Frontend shows **full-width transcript layout** (no split panels)
    - TranscriptBox shows text **without speaker names** (no "Speaker 1:" prefixes)
    - No spinner waiting for speaker identification
    - No summary/analysis tab (chat service not running)

**Check user messaging:**

11. Verify the user can understand WHY they're in transcription-only mode:
    - Service chips clearly indicate diarization and chat are offline
    - Mode label is readable and prominent

Report what you see. Type "approved" if everything works, or describe any issues.
  </how-to-verify>
