
**Candidate 1 — Current baseline (pyannote via WhisperX):**
```python
import whisperx
diarize_model = whisperx.DiarizationPipeline(device="cpu")
diarize_segments = diarize_model(audio_path)
```

**Candidate 2 — diarize library:**
```python
from diarize import diarize
result = diarize(audio_path)
# result.segments, result.num_speakers
```

For each candidate:
1. Measure model load time (cold start)
2. Run diarization 3 times, record RTF and peak memory each run
3. Record number of speakers detected
4. If both produce RTTM output, compare segment overlap quality (not formal DER without reference RTTM, but note agreement/disagreement on speaker count and segment boundaries)
5. Save results to `results/diarization_results.json`
6. Print comparison table

Run candidates sequentially with gc.collect() between runs. Same 12GB memory safety threshold as Plan 1.

Note: Formal DER requires reference RTTM annotations which we likely don't have for wargame audio. Instead, compare:
- Speed (RTF)
- Memory usage
- Speaker count agreement between the two systems
- Qualitative segment boundary alignment

If diarize library fails to install or has compatibility issues, document the error and proceed with pyannote-only results — the recommendation will note diarize as unverified.
