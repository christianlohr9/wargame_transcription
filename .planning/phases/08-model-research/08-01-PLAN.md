  <name>Task 1: Create benchmark harness and prepare test environment</name>
  <files>speaker-diarization-service/benchmarks/benchmark_harness.py, speaker-diarization-service/benchmarks/README.md</files>
  <action>
Create a benchmark harness script that standardizes evaluation across all model candidates. The harness must measure:
- **WER** (Word Error Rate): Compare transcription output against reference text using jiwer library
- **RTF** (Real-Time Factor): elapsed_time / audio_duration (lower = faster)
- **Peak RSS** (memory): Track via psutil before/after model load AND during inference separately
- **Model load time**: Time from cold start to ready-for-inference
- **Audio duration**: For RTF calculation

Install dependencies into the diarization service venv:
```
pip install psutil jiwer
```

For test audio: check if any .wav files exist in the project (wargame recordings from prior testing). If none found, use a short synthetic test by recording silence or downloading a public domain sample (e.g., LibriSpeech test-clean). Create a `benchmarks/test_audio/` directory.

The harness should:
1. Accept a callable (model load + transcribe function) and audio path
2. Run 3 iterations, report mean and std dev for timing metrics
3. Output results as JSON for later aggregation
4. Print a human-readable summary table

Also create a README.md documenting how to run benchmarks and interpret results.

Do NOT build a complex framework — keep it under 150 lines. A single Python file with a `run_benchmark(name, load_fn, transcribe_fn, audio_path, reference_text=None)` function.
  </action>

**Candidate 1 — Current baseline (WhisperX small, int8):**
```python
import whisperx
model = whisperx.load_model("small", device="cpu", compute_type="int8")
result = model.transcribe(audio_path)
```

**Candidate 2 — faster-whisper distil-large-v3 (INT8):**
```python
from faster_whisper import WhisperModel
model = WhisperModel("distil-large-v3", device="cpu", compute_type="int8")
segments, info = model.transcribe(audio_path, beam_size=5)
text = " ".join(seg.text for seg in segments)
```

**Candidate 3 — faster-whisper large-v3-turbo (INT8):**
```python
from faster_whisper import WhisperModel
model = WhisperModel("large-v3-turbo", device="cpu", compute_type="int8")
segments, info = model.transcribe(audio_path, beam_size=5)
text = " ".join(seg.text for seg in segments)
```

For each candidate:
1. Measure model load time (cold start)
2. Run transcription 3 times, record RTF and peak memory each run
3. If reference text available, compute WER; otherwise note "WER: N/A (no reference)"
4. Save all results to `results/transcription_results.json`
5. Print comparison table to stdout

Important: Run each candidate in sequence (not parallel) to avoid memory interference. Explicitly delete model and call gc.collect() + torch.cuda.empty_cache() (if torch available) between candidates to free memory.

Monitor total system memory — if a model would exceed ~12GB RSS (leaving headroom for OS + Docker), skip it with a warning rather than OOM-killing services.

Do NOT test whisper.cpp in this plan — it requires separate tooling (CLI, not Python). Do NOT test Voxtral here — it's a bonus candidate for plan 2 if time permits.
