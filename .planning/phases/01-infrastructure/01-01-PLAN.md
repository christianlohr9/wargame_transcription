<task type="auto">
  <name>Task 1: Add Conductor stack to docker-compose.yml</name>
  <files>blackbox/docker-compose.yml</files>
  <action>
Add the following services to the existing docker-compose.yml:

1. **elasticsearch** service:
   - Image: `docker.elastic.co/elasticsearch/elasticsearch:7.17.11`
   - Port: 9200:9200
   - Environment: `discovery.type=single-node`, `xpack.security.enabled=false`, `ES_JAVA_OPTS=-Xms512m -Xmx512m` (limited for 16GB machine)
   - Volume: `esdata:/var/lib/elasticsearch/data`
   - Healthcheck: `curl -s http://localhost:9200/_cluster/health | grep -q '"status":"green\|yellow"'`

2. **redis** service:
   - Image: `redis:7-alpine`
   - Port: 6379:6379
   - Healthcheck: `redis-cli ping`

3. **conductor-server** service:
   - Image: `conductoross/conductor-standalone:3.15.0`
   - Ports: 8080:8080, 5000:5000 (UI)
   - Environment: Point to redis and elasticsearch
   - depends_on: elasticsearch (healthy), redis (healthy)
   - Healthcheck: `curl -s http://localhost:8080/health | grep -q '"healthy":true'` or `curl -f http://localhost:8080/api/health`

Add `esdata` to the volumes section.

**Important:** Keep existing services (rabbitmq, mongodb, minio) unchanged. Use compose file version 3.8 for health check support. Do NOT use version 2.3.
  </action>
  <verify>docker compose -f blackbox/docker-compose.yml config (validates syntax)</verify>
  <done>docker-compose.yml contains elasticsearch, redis, conductor-server services with health checks</done>
