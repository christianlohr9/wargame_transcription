<task type="auto">
  <name>Task 1: Start infrastructure stack</name>
  <files>blackbox/docker-compose.yml</files>
  <action>
Run docker compose to start all services:

```bash
cd blackbox
docker compose up -d
```

Wait for services to initialize (Conductor takes longer due to Elasticsearch dependency). Check logs for any startup errors:

```bash
docker compose logs --tail=50
```

If any service fails to start, check specific logs:
```bash
docker compose logs conductor-server
docker compose logs elasticsearch
```
  </action>
  <verify>docker compose ps shows all 7 services running (Up status)</verify>
  <done>All services show "Up" or "Up (healthy)" status</done>
</task>

<task type="auto">
  <name>Task 2: Verify service health endpoints</name>
  <files>None (verification only)</files>
  <action>
Test each service is responding:

```bash
# MongoDB
docker exec mongodb mongosh --eval "db.adminCommand('ping')"

# RabbitMQ Management
curl -s http://localhost:15672 | head -1

# MinIO
curl -s http://localhost:9001 | head -1

# Elasticsearch
curl -s http://localhost:9200/_cluster/health | jq .status

# Redis
docker exec redis redis-cli ping

# Conductor API
curl -s http://localhost:8080/health

# Conductor UI (should serve HTML)
curl -s http://localhost:5000 | head -1

# Tika
curl -s http://localhost:9998/version
```

Document any services that fail health checks.
  </action>
  <verify>All 7 curl/exec commands return successful responses</verify>
  <done>Every service responds to its health check endpoint</done>
</task>
