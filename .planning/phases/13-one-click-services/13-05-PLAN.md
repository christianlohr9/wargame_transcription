<objective>
Create Electron desktop app shell with process management for spawning and monitoring Java and Python services.

Purpose: Wrap the entire platform in an Electron app that non-technical users can launch with a double-click. The Electron main process acts as a supervisor that starts, monitors, and stops all backend services.
Output: Working Electron app that loads the Vue frontend, spawns Spring Boot + Python services as child processes, shows service status, and allows toggling optional services.
</objective>

**From RESEARCH.md standard stack:** Electron 34.x, electron-builder 26.x, tree-kill, electron-log, electron-store
**From RESEARCH.md architecture patterns:** Process supervisor, health-check polling, IPC bridge

**Constraining decisions:**
- [13-CONTEXT.md]: Zero-terminal experience — user never sees a console
- [13-CONTEXT.md]: Full offline capability for VS-NfD classified material
- [13-RESEARCH.md]: Use child_process.spawn() with windowsHide: true (not utilityProcess)
- [13-RESEARCH.md]: Use javaw.exe on Windows to avoid console windows
- [Phase 10]: Health checks at trigger time, not startup — services may not be fully ready at boot
- [Phase 11]: Lazy pyannote model loading — first diarization request is slow
- Development on macOS, target deployment on Windows — structure code cross-platform

  <name>Task 1: Create Electron project structure</name>
  <files>
    blackbox-desktop/package.json,

    blackbox-desktop/electron/preload.js,
    blackbox-desktop/electron/processManager.js,

  </files>
  <action>
    1. Create new directory `blackbox-desktop/` at the repo root (sibling to blackbox/, ask-chat-service/, speaker-diarization-service/).

    2. Initialize npm project:
       ```
       cd blackbox-desktop && npm init -y
       npm install electron tree-kill electron-log electron-store
       npm install -D electron-builder
       ```

    3. Create `electron/main.js`:
       - Import ProcessManager and HealthChecker
       - On app.whenReady():
         - Determine resourcesPath (process.resourcesPath in production, relative path in dev)
         - Determine platform-specific binary names:
           - Java: `javaw.exe` on Windows, `java` on macOS/Linux
           - Python: `python.exe` on Windows, `python` on macOS/Linux (or `python3`)
         - Start backend service (Spring Boot JAR) via ProcessManager
         - Create BrowserWindow loading the built frontend (dist/index.html or dev server URL)
       - Expose IPC handlers:
         - `get-service-status` → returns current status of all services
         - `toggle-service` → start/stop optional services (diarization, chat)
         - `get-platform-info` → returns OS, memory, etc.
       - On app.before-quit: call processManager.stopAll()
       - On window-all-closed: quit app (single window app)

    4. Create `electron/preload.js`:
       - Use contextBridge to expose safe IPC API to renderer:
         ```javascript
         contextBridge.exposeInMainWorld('electronAPI', {
           getServiceStatus: () => ipcRenderer.invoke('get-service-status'),
           toggleService: (name, enabled) => ipcRenderer.invoke('toggle-service', name, enabled),
           onServiceStatusChange: (callback) => ipcRenderer.on('service-status-changed', callback),
           getPlatformInfo: () => ipcRenderer.invoke('get-platform-info')
         })
         ```

    5. Create `electron/processManager.js`:
       - ProcessManager class (follow pattern from RESEARCH.md):
         - startService(name, command, args, options): spawn child process with windowsHide: true
         - stopService(name): graceful SIGTERM via tree-kill
         - stopAll(): stop all managed processes
         - getStatus(): return Map of service name → status
         - Handle child process exit events (track status, optional auto-restart)
         - Pipe stdout/stderr to electron-log for debugging
       - Service definitions (configurable paths):
         - backend: java -jar blackbox.jar (port 8081)
         - diarization: python -m uvicorn src.main:app --port 8082 (optional)
         - chat: python -m uvicorn src.main:app --port 8083 (optional)

    6. Create `electron/healthChecker.js`:
       - HealthChecker class:
         - Monitor service health endpoints via HTTP polling
         - Services: backend (localhost:8081/actuator/health), diarization (localhost:8082/health), chat (localhost:8083/health)
         - Polling interval: 5s
         - Emit status changes to main process (which forwards via IPC to renderer)
         - States: starting, healthy, unhealthy, stopped

    7. Update package.json with electron entry point:
       ```json
       {
         "main": "electron/main.js",
         "scripts": {
           "start": "electron .",
           "build": "electron-builder"
         }
       }
       ```

    **Cross-platform design:** Use path.join() everywhere. Use process.platform checks for binary names. Don't hardcode Windows paths. Both macOS (dev) and Windows (prod) should work.

    **Dev mode:** When running `npm start` in development, load frontend from `http://localhost:9003` (Quasar dev server) instead of built files. Use an environment variable or check for dist/index.html existence.

    **Avoid:** Don't try to integrate Electron into the existing frontend project. Keep it as a separate package that wraps the built frontend. Don't use Electron's utilityProcess for Java/Python (it only works for Node.js). Don't add auto-update logic (out of scope per CONTEXT.md).
  </action>

       - Add toggle switches (QToggle) next to diarization and chat service chips:
         - Backend: always on, no toggle (required)
         - Diarization: toggle on/off (optional service)
         - Chat: toggle on/off (optional service)
       - Toggle switches only visible when running in Electron (isElectron check)
       - Show clear status: "Running" (green), "Starting..." (yellow), "Stopped" (grey), "Error" (red)

    4. Add TypeScript interface for window.electronAPI:
       ```typescript
       declare global {
         interface Window {
           electronAPI?: {
             getServiceStatus: () => Promise<Record<string, string>>;
             toggleService: (name: string, enabled: boolean) => Promise<void>;
             onServiceStatusChange: (callback: (event: any, status: Record<string, string>) => void) => void;

         }
       }
       ```
