<task type="auto">
  <name>Task 1: Create jlink custom JRE build script</name>
  <files>
    blackbox-desktop/scripts/build-java-runtime.sh,
    blackbox-desktop/scripts/build-java-runtime.md
  </files>
  <action>
    1. Create `blackbox-desktop/scripts/build-java-runtime.sh`:

    The script should:
    a. First build the Spring Boot fat JAR:
       ```bash
       cd blackbox/backend && mvn clean package -DskipTests
       ```
    b. Find the built JAR:
       ```bash
       JAR_PATH=$(find blackbox/backend/blackbox_application/target -name "*.jar" -not -name "*-sources.jar" | head -1)
       ```
    c. Detect required Java modules using jdeps:
       ```bash
       jdeps --ignore-missing-deps --multi-release 21 --print-module-deps "$JAR_PATH"
       ```
       Note: Spring Boot fat JARs may not work directly with jdeps. May need to extract and analyze individual JARs, or use a known module set for Spring Boot apps. Common required modules: java.base, java.desktop, java.instrument, java.management, java.naming, java.net.http, java.prefs, java.scripting, java.security.jgss, java.security.sasl, java.sql, java.xml, jdk.crypto.ec, jdk.httpserver, jdk.unsupported

    d. Generate custom JRE with jlink:
       ```bash
       jlink --add-modules <detected-modules> \
             --compress=2 \
             --no-header-files \
             --no-man-pages \
             --strip-debug \
             --output blackbox-desktop/resources/runtime/java
       ```

    e. Copy the Spring Boot JAR to resources:
       ```bash
       cp "$JAR_PATH" blackbox-desktop/resources/app/blackbox.jar
       ```

    f. Print summary: JRE size, JAR size, total.

    2. Make the script executable: `chmod +x blackbox-desktop/scripts/build-java-runtime.sh`

    3. Create `blackbox-desktop/scripts/build-java-runtime.md` documenting:
       - Prerequisites: JDK 21 installed, Maven installed
       - For macOS development: generates macOS JRE (dev/test only)
       - For Windows production: must run on Windows with JDK 21 to get Windows JRE
       - Troubleshooting: common jdeps issues with fat JARs

    4. Update processManager.js to use the bundled JRE path:
       ```javascript
       const javaPath = path.join(resourcesPath, 'runtime', 'java', 'bin',
         process.platform === 'win32' ? 'javaw.exe' : 'java');
       const jarPath = path.join(resourcesPath, 'app', 'blackbox.jar');
       ```

    **Important:** The jlink JRE is platform-specific. A macOS build produces a macOS JRE. For the Windows target, the script must be run on Windows. Document this clearly.

    **Avoid:** Don't try to cross-compile JREs (jlink doesn't support this natively). Don't include jdk.* modules unless specifically needed (keeps size down). Don't include java.desktop unless Spring Boot actually requires it (check jdeps output).
  </action>
  <verify>

    `blackbox-desktop/resources/runtime/java/bin/java -version` prints Java 21.
    `ls blackbox-desktop/resources/app/blackbox.jar` exists.
  </verify>

<task type="auto">
  <name>Task 2: Create conda-pack Python environment build script and model bundling</name>
  <files>
    blackbox-desktop/scripts/build-python-runtime.sh,
    blackbox-desktop/scripts/build-python-runtime.md,
    blackbox-desktop/scripts/bundle-models.sh
  </files>
  <action>
    1. Create `blackbox-desktop/scripts/build-python-runtime.sh`:

    The script should:
    a. Create a conda environment with all diarization + chat dependencies:
       ```bash
       conda create -n blackbox-python python=3.12 -y
       conda activate blackbox-python
       # Install PyTorch CPU-only (lighter than GPU version)
       pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
       # Install from requirements
       pip install -r speaker-diarization-service/requirements.txt
       pip install -r ask-chat-service/requirements.txt
       # Remove conductor-python if still in requirements (should be gone from Plan 02)
       ```
    b. Pack the environment:
       ```bash
       conda pack -n blackbox-python -o blackbox-desktop/resources/runtime/python-env.tar.gz
       ```
    c. Extract to runtime directory:
       ```bash
       mkdir -p blackbox-desktop/resources/runtime/python
       tar -xzf blackbox-desktop/resources/runtime/python-env.tar.gz -C blackbox-desktop/resources/runtime/python
       ```
    d. Run conda-unpack to fix paths:
       ```bash
       cd blackbox-desktop/resources/runtime/python && source bin/activate && conda-unpack && deactivate
       ```
    e. Copy Python service source code:
       ```bash
       cp -r speaker-diarization-service/src blackbox-desktop/resources/app/diarization/
       cp -r ask-chat-service/src blackbox-desktop/resources/app/chat/
       ```
    f. Print summary: Python env size, total size.

    2. Create `blackbox-desktop/scripts/bundle-models.sh`:

    Pre-download ML models so the app works offline (air-gapped):
    a. Download faster-whisper large-v3-turbo model:
       ```bash
       python -c "from faster_whisper import WhisperModel; WhisperModel('deepdml/faster-whisper-large-v3-turbo-ct2', device='cpu', compute_type='int8')"
       ```
       This downloads to ~/.cache/huggingface/. Copy the cached model:
       ```bash
       cp -r ~/.cache/huggingface/hub/models--deepdml--faster-whisper-large-v3-turbo-ct2 blackbox-desktop/resources/models/whisper/
       ```
    b. Download pyannote models (requires HF_TOKEN):
       ```bash
       python -c "from pyannote.audio import Pipeline; Pipeline.from_pretrained('pyannote/speaker-diarization-3.1', use_auth_token='$HF_TOKEN')"
       ```
       Copy cached models:
       ```bash
       cp -r ~/.cache/huggingface/hub/models--pyannote--* blackbox-desktop/resources/models/pyannote/
       ```
    c. Set environment variable in processManager to point to bundled models:
       ```
       HF_HOME=<resourcesPath>/models
       WHISPER_CACHE=<resourcesPath>/models/whisper
       ```

    3. Create `blackbox-desktop/scripts/build-python-runtime.md` documenting:
       - Prerequisites: conda/miniconda installed
       - For macOS: generates macOS Python env (dev/test only)
       - For Windows production: MUST run on Windows x64 (conda-pack is platform-specific)
       - Model bundling: requires HF_TOKEN for pyannote models, internet access for download
       - Total expected size: ~2-3GB for runtime + models

    4. Update processManager.js to use bundled Python path:
       ```javascript
       const pythonPath = path.join(resourcesPath, 'runtime', 'python',
         process.platform === 'win32' ? 'python.exe' : 'bin/python');
       ```

    **Important:** Like jlink, conda-pack is platform-specific. macOS build produces macOS Python. Document that production build must happen on Windows. The model download happens once during build, not at runtime (critical for air-gapped deployment).

    **Avoid:** Don't install GPU PyTorch (adds ~2GB for CUDA that won't be used on CPU-only target). Don't include test/dev dependencies. Don't try to cross-pack (build on macOS for Windows — won't work).
  </action>
