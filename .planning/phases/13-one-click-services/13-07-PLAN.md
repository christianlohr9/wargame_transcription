  <action>
    1. Create `blackbox-desktop/electron-builder.yml`:
       ```yaml
       appId: com.blackbox.wargaming
       productName: Blackbox Wargaming
       directories:
         output: dist-electron

       # macOS target (for development testing)
       mac:
         target:
           - target: dir
             arch: [x64, arm64]
         icon: build/icon.icns

       # Windows target (for production)
       win:
         target:
           - target: nsis
             arch: [x64]
         icon: build/icon.ico

       nsis:
         oneClick: false
         perMachine: false
         allowToChangeInstallationDirectory: true
         deleteAppDataOnUninstall: false

       extraResources:
         - from: resources/runtime/java
           to: runtime/java
           filter: ["**/*"]
         - from: resources/runtime/python
           to: runtime/python
           filter: ["**/*"]
         - from: resources/models
           to: models
           filter: ["**/*"]
         - from: resources/app
           to: app
           filter: ["**/*"]

       asar: true
       asarUnpack:
         - "**/*.node"
       ```

    2. Update package.json scripts:
       ```json
       {
         "scripts": {
           "start": "electron .",
           "build:frontend": "cd ../blackbox/frontend && npx quasar build",
           "build:mac": "electron-builder --mac --dir",
           "build:win": "electron-builder --win",
           "build:all": "bash scripts/build-app.sh"
         }
       }
       ```

    3. Create `blackbox-desktop/scripts/build-app.sh` — full build pipeline:
       ```bash
       #!/bin/bash
       set -e

       echo "=== Building Blackbox Wargaming Desktop App ==="

       # Step 1: Build frontend
       echo "[1/5] Building frontend..."
       cd ../blackbox/frontend && npx quasar build
       cp -r dist/spa ../../blackbox-desktop/dist

       # Step 2: Build Java backend + runtime
       echo "[2/5] Building Java runtime..."
       cd ../../blackbox-desktop
       bash scripts/build-java-runtime.sh

       # Step 3: Build Python runtime (if conda available)
       echo "[3/5] Building Python runtime..."
       if command -v conda &> /dev/null; then
         bash scripts/build-python-runtime.sh
       else
         echo "WARNING: conda not available, skipping Python runtime build"
       fi

       # Step 4: Bundle models (if requested)
       if [ "$BUNDLE_MODELS" = "true" ]; then
         echo "[4/5] Bundling ML models..."
         bash scripts/bundle-models.sh
       else
         echo "[4/5] Skipping model bundling (set BUNDLE_MODELS=true to include)"
       fi

       # Step 5: Build Electron app
       echo "[5/5] Building Electron app..."
       if [ "$(uname)" = "Darwin" ]; then
         npx electron-builder --mac --dir
       else
         npx electron-builder --win
       fi

       echo "=== Build complete! ==="
       echo "Output: blackbox-desktop/dist-electron/"
       ```

    4. Create placeholder icon files:
       - `blackbox-desktop/build/icon.icns` (macOS) — placeholder, can be replaced later
       - `blackbox-desktop/build/icon.ico` (Windows) — placeholder
       Use a simple placeholder or copy from existing assets. Create a 512x512 PNG and convert if no icons exist.

    5. Create a `blackbox-desktop/.gitignore`:
       ```
       node_modules/
       dist/
       dist-electron/
       resources/runtime/
       resources/models/
       resources/app/
       ```
       The runtime/models/app directories are build artifacts, not source code.

    **Avoid:** Don't configure code signing (requires organization certificate — open question from RESEARCH.md). Don't add auto-update (out of scope). Don't try to build Windows installer on macOS (electron-builder can cross-compile but it's unreliable for NSIS).
  </action>
