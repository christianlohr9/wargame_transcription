<task type="auto">
  <name>Task 1: Install export libraries and create transcript export composable</name>
  <files>
    blackbox/frontend/package.json,
    blackbox/frontend/src/composables/useTranscriptExport.ts
  </files>
  <action>
    1. Install dependencies:
       ```
       cd blackbox/frontend && npm install pdfmake docx file-saver
       npm install -D @types/pdfmake @types/file-saver
       ```

    2. Read TranscriptBox.vue and models.ts to understand the transcript data structure. Identify what fields are available: segments/fragments with timestamp, speaker name, text content.

    3. Create composable `src/composables/useTranscriptExport.ts` with three export functions:

    **exportPdf(transcript, filename):**
    - Use pdfmake with declarative document definition
    - Layout: timestamp column (80px) | speaker column (100px) | text content (fill)
    - Title: workspace name or "Transcript"
    - Style: monospace timestamps (grey), bold speaker names, normal text
    - Include generation date in footer
    - Call pdfMake.createPdf(docDefinition).download(filename)

    **exportDocx(transcript, filename):**
    - Use docx library with Document/Paragraph/TextRun
    - Same layout concept: [timestamp] Speaker: text
    - Use Packer.toBlob() + saveAs() from file-saver
    - Include document title and metadata

    **exportTxt(transcript, filename):**
    - Simple string concatenation: `[timestamp] Speaker: text\n`
    - Create Blob with text/plain charset=utf-8
    - Use saveAs() from file-saver

    Each function should accept the transcript data in whatever format TranscriptBox.vue uses and a filename string.

    **Handle edge cases:**
    - If no speaker labels (transcription_only mode): omit speaker from output
    - If timestamps missing: omit timestamp column
    - Empty transcript: show message in exported document

    **Avoid:** Don't add backend endpoints for export. Don't import the full pdfmake build if tree-shakeable (check). Don't add export for analysis/summary data — just the transcript for now.
  </action>
  <verify>
    `cd blackbox/frontend && npm run build` (or `npx quasar build`) succeeds without errors.
    `ls blackbox/frontend/src/composables/useTranscriptExport.ts` exists.
    `grep -r "pdfmake\|docx\|file-saver" blackbox/frontend/package.json` shows all three dependencies.
  </verify>
  <done>
