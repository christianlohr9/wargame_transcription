def faster_whisper_distil_large_v35_load():
    from faster_whisper import WhisperModel

Modify WhisperXDiarizationService to use faster-whisper directly for transcription instead of WhisperX's bundled model. Use the benchmark winner from Task 1 (v3.5 if it passed, v3 as fallback).

**Changes to `__init__`:**
Replace:
```python
self.whisper_model = whisperx.load_model("small", self.device, compute_type=self.compute_type)
```
With:
```python
from faster_whisper import WhisperModel
self.whisper_model = WhisperModel(
    "[winner model name]",  # e.g. "deepdml/faster-distil-whisper-large-v3.5" or "distil-large-v3"
    device=self.device,
    compute_type=self.compute_type
)
```

**Changes to `diarize` method:**
Replace the transcription call:
```python
result = self.whisper_model.transcribe(audio, batch_size=self.batch_size, language=language)
```
With faster-whisper transcription + segment format adapter:
```python
# Transcribe with faster-whisper
segments_gen, info = self.whisper_model.transcribe(
    audio, beam_size=5, language=language
)
# Convert faster-whisper Segment objects to WhisperX dict format
segments = [
    {"start": seg.start, "end": seg.end, "text": seg.text}
    for seg in segments_gen
]
detected_language = language or info.language
result = {"segments": segments, "language": detected_language}
```

**Changes to alignment call:**
Update the language code reference:
```python
model_a, metadata = whisperx.load_align_model(
    language_code=result["language"],
    device=self.device
)
```
This line stays the same since we're constructing `result` with the same dict format.

**What NOT to change:**
- `self.diarize_pipeline` initialization — unchanged
- `DiarizationPipeline` call — unchanged
- `whisperx.align()` call — unchanged (it receives same dict format)
- `whisperx.assign_word_speakers()` — unchanged
- `_map_to_model()` — unchanged
- `_get_hf_token()` — unchanged
- Audio loading with `whisperx.load_audio()` — unchanged

**Critical pitfall (from RESEARCH.md):** The faster-whisper `transcribe()` returns a GENERATOR. The list comprehension consumes it immediately into a list. Do NOT try to iterate the generator twice.

**Critical pitfall (from RESEARCH.md):** faster-whisper accepts numpy ndarray (1D float32 16kHz) as the `audio` parameter — same format as `whisperx.load_audio()` output. If this fails at runtime, fall back to passing the `audio_file` path string instead. But the numpy array should work since the benchmark already confirms faster-whisper handles audio correctly.

Remove `self.batch_size` instance variable since faster-whisper uses `beam_size` instead. Update the docstring to reflect the hybrid pipeline architecture.
