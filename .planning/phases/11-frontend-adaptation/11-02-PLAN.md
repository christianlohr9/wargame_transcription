```typescript
const hasSpeakerData = computed(() =>
  workspace.value?.determinedSpeakers != null &&
  workspace.value.determinedSpeakers.length > 0
)

const hasRoundsData = computed(() =>
  workspace.value?.determinedRounds != null &&
  workspace.value.determinedRounds.length > 0
)

const hasAnalyticsData = computed(() =>
  hasSpeakerData.value && speakerStats.value.length > 0

const hasLeftPanel = computed(() =>
  hasRoundsData.value || hasAnalyticsData.value
```

**TranscriptBox changes:**

1. The transcript timeline entries currently show speaker names via `translateSpeakerName()`. In transcription_only mode, `determinedSpeakers` is null/empty, so `translateSpeakerName` falls through and returns the original generic "Speaker" label.

2. Hide the speaker name spinner: the current template shows `<q-spinner-dots v-if="!workspace?.determinedSpeakers" />` next to speaker names. This spinner implies speakers are loading when in reality they won't come in transcription_only mode. Change the condition to only show the spinner when transcription exists but speakers are still expected (i.e., fragments have distinct speaker labels other than just "Speaker"):

```typescript
const expectsSpeakers = computed(() => {
  if (!props.workspace?.transcription?.fragments?.length) return false
  if (props.workspace.determinedSpeakers) return false // already resolved
  const speakers = new Set(props.workspace.transcription.fragments.map(f => f.speaker))
  return speakers.size > 1 || !speakers.has('Speaker')
})
```

Use `v-if="expectsSpeakers"` on the spinner instead of `v-if="!workspace?.determinedSpeakers"`.

3. When all fragments have the same generic "Speaker" label (transcription_only), hide the speaker name entirely from the timeline entries to avoid showing "Speaker" on every single entry. Add a computed:

```typescript
const showSpeakerNames = computed(() => {
  const fragments = props.workspace?.transcription?.fragments ?? []
  const speakers = new Set(fragments.map(f => f.speaker))
  return speakers.size > 1 || (speakers.size === 1 && !speakers.has('Speaker'))
})
```

Wrap the speaker name `<div class="text-h5">` in `v-if="showSpeakerNames"`.

**AnalIntelligence changes:**

The component already gates on `hasTranscript`. No changes needed — when IndexPage doesn't render AnalIntelligence (because no speaker data), this component is simply not mounted. The existing `hasTranscript` check is a secondary safeguard that can remain as-is.

Keep all changes minimal. Do not refactor unrelated code.
