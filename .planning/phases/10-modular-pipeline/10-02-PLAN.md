       - In _execute_unsafe(), extract pipeline_mode from task.input_data (default to "full" if not present for backwards compatibility)
       - Pass pipeline_mode to diarization_service.diarize()

    2. In the diarization_service module (__init__.py or wherever `diarize` is exposed):
       - Update the diarize function signature to accept `pipeline_mode: str = "full"` parameter
       - Pass it through to the WhisperXDiarizationService.diarize() call

    3. In WhisperXDiarizationService (whisperx_diarization_service.py):
       - Add `pipeline_mode: str = "full"` parameter to diarize() method
       - After transcription (step 2) and alignment (step 3), check mode:
         - If pipeline_mode == "transcription_only": SKIP steps 4 and 5 (pyannote diarization and speaker assignment). Instead, map segments directly to DiarizationModel with a generic "Speaker" label for all fragments. This means transcription_only mode returns consistent output format (same fields) but with a single generic speaker.
         - If pipeline_mode is "full" or "transcription_diarization": run the full pipeline as-is (diarization + speaker assignment)
       - For transcription_only, create a separate `_map_transcription_only()` method that converts aligned segments to DiarizationModel with speaker="Speaker" for all fragments. Keep the same output contract: audio_duration, available_speakers (["Speaker"]), fragments with speaker/transcription/start_time/end_time.

    **Why skip diarization entirely for transcription_only:** pyannote loads and processes for ~10-15 seconds on CPU. Skipping it makes transcription_only mode feel genuinely faster per CONTEXT.md: "should feel lighter and faster, not just full pipeline minus features."

    **Why keep alignment for transcription_only:** WhisperX alignment gives word-level timestamps which improve fragment quality. It's fast (~1-2 seconds) and the benefit is worth it.

    AVOID: Don't change the DiarizationModel schema — output contract must be consistent across all modes per RESEARCH.md pitfall #4. Don't make pipeline_mode a required parameter — default to "full" for backward compatibility with existing direct API calls. Don't skip alignment — it's cheap and improves quality.
