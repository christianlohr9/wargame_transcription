<task type="auto">
  <name>Task 1: Initialize electron-store with LLM settings and IPC handlers</name>
  <files>blackbox-desktop/electron/main.js, blackbox-desktop/electron/preload.js</files>
  <action>
1. In `main.js`:
   - Import and initialize electron-store with schema/defaults:
     ```
     llmBackend: { type: 'string', default: 'llamacpp' }
     ollamaEndpoint: { type: 'string', default: 'http://localhost:11434' }
     ollamaModel: { type: 'string', default: '' }
     ```
   - Add IPC handlers:
     - `get-settings`: Returns all settings from store as object
     - `set-setting`: Takes {key, value}, validates key exists in schema, stores it
     - `get-setting`: Takes key string, returns single value
   - When `set-setting` changes `llmBackend`, stop and restart the chat service with new env vars (call processManager.stopService('chat') then processManager.startService('chat'))

2. In `preload.js`:
   - Expose settings IPC through contextBridge:
     - `getSettings()` → invoke 'get-settings'
     - `setSetting(key, value)` → invoke 'set-setting'
     - `getSetting(key)` → invoke 'get-setting'

AVOID: Storing sensitive data (API keys) in electron-store without encryption. For this phase, only store backend selection and Ollama endpoint/model — no secrets.
  </action>
  <verify>Inspect main.js for electron-store initialization and IPC handlers. Inspect preload.js for exposed settings API.</verify>
  <done>electron-store initialized with LLM defaults, IPC handlers for get/set settings work, preload exposes settings API to renderer</done>
</task>
