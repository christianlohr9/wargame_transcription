<tasks>

<task type="auto">
  <name>Task 1: Create .env file with local configuration</name>
  <files>blackbox/.env</files>
  <action>
Create .env file in blackbox/ directory based on .env.template with these values:

```
SPRING_PROFILES_ACTIVE=local
SERVER_PORT=8081
SERVER_SERVICE_URL=http://localhost:8081
BACKEND_TIKA_BASE_URL=http://localhost:9998
BACKEND_CONDUCTOR_BASE_URL=http://localhost:8080/api
BACKEND_ASK_CHAT_BASE_URL=http://localhost:8000
```

Key changes from template:
- Add SERVER_PORT=8081 (avoid conflict with Conductor on 8080)
- Update SERVER_SERVICE_URL to use port 8081
- Add BACKEND_ASK_CHAT_BASE_URL for future use (chat service will run on 8000)

Do NOT modify application.yml - it already has correct variable placeholders with defaults.
  </action>
  <verify>cat blackbox/.env shows all 6 environment variables with correct values</verify>
  <done>.env file exists with correct local configuration</done>
</task>

<task type="auto">
  <name>Task 2: Build and run backend with Maven</name>
  <files>N/A (no file changes)</files>
  <action>
1. Ensure Docker infrastructure is running (docker compose should show 7 healthy containers)
2. Build the backend: `cd blackbox && ./mvnw clean package -DskipTests`
3. Start the backend in background: `cd blackbox && source .env && ./mvnw spring-boot:run -pl backend/blackbox_application &`
4. Wait for startup (watch for "Started Application" in logs)
5. Verify connectivity:
   - Backend health: curl http://localhost:8081/actuator/health (or /swagger-ui.html)
   - MongoDB: Check logs for successful MongoDB connection
   - Conductor: Check logs for Conductor client initialization

If build fails with frontend copy error (frontend not built yet), that's expected - the backend can still run without embedded frontend since we'll use dev server later.
  </action>
  <verify>curl -s http://localhost:8081/swagger-ui.html returns 200, docker logs show no connection errors</verify>
  <done>Backend running on port 8081 with successful connections to MongoDB and Conductor</done>
</task>
