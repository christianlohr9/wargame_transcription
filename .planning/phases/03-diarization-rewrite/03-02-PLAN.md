**Prior decisions:**
- Backend on port 8081 (avoids Conductor 8080 conflict)
- Diarization service on port 8080 (in main.py)
- CPU-only inference with int8 compute type
- Batch processing acceptable (overnight jobs)

**From research:**
- First run downloads models (~2GB for small whisper + alignment + diarization)
- Model loading takes 30-60 seconds
- Expect ~10x real-time for CPU processing (1-min audio = ~10 min)
- HF_TOKEN required for pyannote models (must accept license on HuggingFace)

<task type="auto">
  <name>Task 1: Create .env and install dependencies</name>
  <files>speaker-diarization-service/.env</files>
  <action>
1. Create .env file from template:
   - Copy .env.template to .env
   - Set HF_TOKEN (user must have accepted pyannote model license on HuggingFace)
   - Set CONDUCTOR_URL=http://localhost:8080

2. Create Python virtual environment and install dependencies:
   ```bash
   cd speaker-diarization-service
   python3 -m venv venv
   source venv/bin/activate  # or venv\Scripts\activate on Windows
   pip install -r requirements.txt
   ```

3. Verify ffmpeg is available:
   ```bash
   ffmpeg -version
   ```
   If not installed: `brew install ffmpeg` (macOS) or `apt install ffmpeg` (Linux)

Note: First pip install will be slow as it downloads ~2GB of PyTorch and model files.
Note: User needs a HuggingFace account and must accept the pyannote model licenses at:
- https://huggingface.co/pyannote/speaker-diarization-3.1
- https://huggingface.co/pyannote/segmentation-3.0
  </action>
  <verify>pip list | grep -E "whisperx|torch" shows both installed</verify>
  <done>.env created, venv created, all dependencies installed including whisperx and torch</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Accept pyannote model licenses on HuggingFace</action>
  <instructions>
WhisperX requires pyannote models which are gated on HuggingFace. You need to:

1. Create a HuggingFace account if you don't have one: https://huggingface.co/join
2. Visit and accept the license for these models:
   - https://huggingface.co/pyannote/speaker-diarization-3.1 (click "Agree and access repository")
   - https://huggingface.co/pyannote/segmentation-3.0 (click "Agree and access repository")
3. Create an access token at: https://huggingface.co/settings/tokens
4. Update speaker-diarization-service/.env with: HF_TOKEN=hf_your_token_here
  </instructions>
  <verification>The .env file has HF_TOKEN set to a valid token starting with "hf_"</verification>
  <resume-signal>Type "done" when you have accepted the licenses and updated .env with your HF_TOKEN</resume-signal>

<task type="auto">
  <name>Task 2: Start diarization service and verify health</name>
  <files>speaker-diarization-service/src/main.py</files>
  <action>
1. Ensure Docker infrastructure is running (from Phase 1):
   ```bash
   cd blackbox && docker compose up -d
   ```

2. Start the diarization service:
   ```bash
   cd speaker-diarization-service
   source venv/bin/activate
   PYTHONPATH=src python -m uvicorn main:app --host 0.0.0.0 --port 8082 --reload
   ```

   Note: Using port 8082 to avoid conflict with Conductor (8080) and backend (8081).
   Note: First startup will be SLOW (30-60s) as models load.

3. Verify health endpoint responds:
   ```bash
   curl http://localhost:8082/health
   ```

If startup fails with import errors, check PYTHONPATH includes src directory.
If startup fails with HF token error, verify .env has valid HF_TOKEN.
  </action>
  <verify>curl http://localhost:8082/health returns 200</verify>
  <done>Diarization service running on port 8082, health endpoint responding</done>
</task>
